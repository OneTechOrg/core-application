# HIST-2026-024: Swagger/OpenAPI para APIs do RappiDrive

**Data**: 09/03/2026  
**Status**: ✅ Concluído  
**Tipo**: Melhoria Técnica  
**Tamanho**: Pequeno (1 dia)  
**Prioridade**: Média

## 1. Problema

A API já possui endpoints funcionais, mas a documentação interativa ainda não está padronizada para todos os recursos de negócio. Isso dificulta:
- descoberta rápida dos endpoints;
- alinhamento entre backend, mobile e integrações;
- validação de contratos em ambientes de teste.

## 2. Objetivo

Disponibilizar documentação Swagger/OpenAPI completa e consistente para os endpoints principais do projeto.

## 3. Escopo (Pequeno)

- Configurar e validar exposição de `/swagger-ui` e `/v3/api-docs`.
- Garantir metadados globais da API (título, versão, descrição, segurança JWT).
- Documentar endpoints críticos com `@Operation`, `@ApiResponses` e schemas de request/response.
- Padronizar exemplos mínimos para fluxos: `trips`, `drivers`, `payments`, `notifications`.

## 4. Critérios de Aceite

- Acesso ao Swagger UI funcionando em ambiente local.
- OpenAPI JSON válido e gerado sem erros.
- Endpoints com autenticação documentam claramente necessidade de Bearer Token.
- Pelo menos os controllers de maior uso possuem descrição e respostas HTTP esperadas.
- Não expor endpoints internos/sensíveis sem necessidade.

## 5. Entregáveis

1. Ajustes na configuração OpenAPI/Swagger da camada `infrastructure`.
2. Anotações de documentação nos controllers priorizados.
3. Revisão de segurança para não abrir documentação indevidamente em produção.
4. Atualização de documentação em `docs/` com instruções de uso.

## 6. Definição de Pronto (DoD)

- [ ] Build e testes passando.
- [x] Swagger UI navegável com endpoints documentados.
- [ ] Contratos principais revisados pelo time.
