# HIST-2026-027: Integração com Stripe — Adapter de Gateway de Pagamento

**Data**: 30/04/2026
**Status**: 🔲 Pendente
**Tipo**: Feature
**Tamanho**: Pequeno (1–2 dias)
**Prioridade**: Alta

---

## 1. Contexto

O `PaymentGatewayPort` já existe como abstração limpa em `application/ports/output/`. Atualmente a única implementação é `MockPaymentGatewayAdapter`, que simula pagamentos com 95% de taxa de sucesso e está anotada com `@Component` sem restrição de profile — ou seja, rodaria em produção sem alteração.

A integração com Stripe é o próximo passo natural para viabilizar pagamentos reais. O port foi desenhado exatamente para isso: trocar o adapter sem tocar no domínio ou nos use cases.

Há, porém, um gap técnico que precisa ser resolvido antes de implementar o adapter: o Stripe não recebe dados brutos de cartão no servidor — ele exige um **gateway token** (`pm_xxxxx`) gerado pelo Stripe.js ou pelo SDK mobile no lado do cliente. Esse token é opaco e precisa fluir do DTO de entrada até o `PaymentGatewayPort.PaymentGatewayRequest`.

## 3. Design técnico

### 3.1 Campo `gatewayToken`

O `PaymentMethod` do domínio armazena apenas metadados (últimos 4 dígitos, bandeira, chave Pix). O token do Stripe é infraestrutura — não entra no domínio.

O campo `gatewayToken` é adicionado como `String` nullable nos records de comando e no `PaymentGatewayRequest`. É opaco para a camada de aplicação: apenas o adapter sabe que é um `pm_xxxxx` do Stripe.

```
ProcessPaymentRequest.stripePaymentMethodId
  → ProcessPaymentCommand.gatewayToken
    → PaymentGatewayPort.PaymentGatewayRequest.gatewayToken
      → StripePaymentGatewayAdapter (usa como PaymentMethod ID)
```

Para pagamentos em dinheiro (CASH) ou PIX (nesta fase), `gatewayToken` é `null` e o adapter não é chamado (fluxo já existente em `ProcessPaymentUseCase`).

### 3.2 `StripePaymentGatewayAdapter` e Idempotência

O adapter deve utilizar o `tripId` como `Idempotency-Key` nas requisições ao Stripe para garantir que falhas de rede não resultem em cobranças duplicadas.

```java
RequestOptions options = RequestOptions.builder()
    .setIdempotencyKey(request.tripId().toString())
    .build();
```

### 3.3 Tratamento de Exceções

O adapter deve traduzir `StripeException` para `PaymentProcessingException`:
- `CardException`: Erro de negócio (cartão recusado, saldo insuficiente).
- `ApiConnectionException` / `RateLimitException`: Erros transientes que ativam o retry/circuit breaker.
- Outras: Erros de infraestrutura mapeados para falha de pagamento.

### 3.4 `StripeConfiguration`

A chave da API é injetada via variável de ambiente `STRIPE_API_KEY`. Nunca hardcoded.
