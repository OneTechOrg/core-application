# HIST-2026-025: Hardening de Segurança — Tenant Isolation e Autenticação

**Data**: 24/04/2026  
**Status**: ✅ Concluído  
**Tipo**: Melhoria de Segurança  
**Tamanho**: Médio (2–3 dias)  
**Prioridade**: Alta

---

## 1. Contexto

Uma análise de segurança foi conduzida sobre os fluxos de multi-tenancy e autenticação implementados nos últimos commits (HIST-2026-019 a HIST-2026-024). Nenhuma vulnerabilidade crítica explorável foi identificada — o sistema já possui três camadas de isolamento sobrepostas:

1. **HTTP** — `TenantResolverFilter` valida presença e existência do tenant no banco antes de prosseguir.
2. **Aspecto AOP** — `TenantFilterAspect` habilita o `@Filter` do Hibernate em todo repositório JPA.
3. **Banco de dados** — Todas as entidades carregam `@Filter(condition = "tenant_id = :tenantId")`, gerando `WHERE tenant_id = ?` em toda query.

Contudo, foram identificadas três oportunidades de **defesa em profundidade** que devem ser endereçadas para maturidade do produto.

## 2. Achados e Melhorias

### 2.1 Validação cruzada da claim JWT com o header `X-Tenant-ID`

**Arquivo**: `TenantResolverFilter.java` — bloco TODO nas linhas 116–127  
**Risco atual**: Baixo (mitigado pelo Hibernate filter)  
**Risco residual**: Um usuário autenticado no Tenant A pode enviar `X-Tenant-ID: B`. O banco retornará dados do Tenant A (filtro Hibernate) mas o log de auditoria registrará acesso ao Tenant B, criando inconsistência rastreável.

**Melhoria**: Implementar a validação pendente no TODO:
```java
// Extrair claim tenant_id do JWT e verificar que coincide com X-Tenant-ID
String jwtTenantId = extractTenantClaimFromJwt(authentication);
if (!jwtTenantId.equals(resolvedTenantId.asString())) {
    sendErrorResponse(response, HttpServletResponse.SC_FORBIDDEN, "Tenant mismatch");
    return;
}
```

---

### 2.2 Correspondência de caminhos no bypass de admin (`/api/admin/`)

**Arquivo**: `TenantResolverFilter.java` — método `isPublicEndpoint()`, linha ~148  
**Risco atual**: Baixo (Spring Security exige autenticação em todos os endpoints)  
**Risco residual**: O método usa `requestUri.startsWith(endpoint)` para `/api/admin/`, o que contornaria a validação de tenant para rotas inexistentes como `/api/admin-tools/` ou `/api/administrators/`. Embora essas rotas não existam hoje, é um padrão frágil.

**Melhoria**: Substituir `startsWith` por correspondência de caminho precisa:
```java
// Atual (frágil):
return PUBLIC_ENDPOINTS.stream().anyMatch(requestUri::startsWith);

// Proposto (preciso):
AntPathMatcher matcher = new AntPathMatcher();
return PUBLIC_ENDPOINTS.stream().anyMatch(p -> matcher.match(p, requestUri));
// e alterar "/api/admin/" para "/api/admin/**" no conjunto de endpoints públicos
```

---

### 2.3 Anotações explícitas de autorização nos controllers de negócio

**Arquivos**: `PaymentController.java`, `NotificationController.java`  
**Risco atual**: Baixo (isolamento garantido pelo Hibernate filter)  
**Risco residual**: A ausência de `@PreAuthorize` dificulta auditoria de segurança, torna o contrato implícito e facilita regressões se a camada de filtro for refatorada.

**Melhoria**: Adicionar `@PreAuthorize` para documentar e enforçar intenção:
```java
// PaymentController — endpoint cross-tenant para admins
@PreAuthorize("hasRole('ADMIN') or @tenantSecurity.belongsTo(authentication, #tenantId)")
public ResponseEntity<List<PaymentResponse>> getPaymentsByTenantAndPeriod(
    @PathVariable UUID tenantId, ...)

// NotificationController — endpoints por usuário autenticado
@PreAuthorize("isAuthenticated()")
public ResponseEntity<Page<NotificationResponse>> getUserNotifications(...)
```

## 3. Critérios de Aceite

- [x] `TenantResolverFilter` extrai e valida a claim `tenant_id` do JWT contra o header `X-Tenant-ID`; retorna `403` em caso de divergência.
- [x] Método `isPublicEndpoint()` usa `AntPathMatcher` em vez de `startsWith`; bypass de admin restrito a `/api/admin/**` com padrão glob.
- [x] `PaymentController.getPaymentsByTenantAndPeriod()` anotado com `@PreAuthorize("hasRole('ADMIN')")`.
- [x] Endpoints de `NotificationController` anotados com `@PreAuthorize` (`ADMIN` para send, `isAuthenticated()` para consultas/leitura).
- [x] `mvn test` verde (ArchUnit 21 regras + 63 testes unitários).
- [ ] `mvn verify` verde (integração com Testcontainers) — requer Docker.
- [x] Nenhuma nova violação de camada introduzida.
