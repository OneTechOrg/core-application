# HIST-2026-039: Fechamento do Loop de Notificações de Corrida (TRIP_ACCEPTED, TRIP_STARTED, TRIP_COMPLETED, TRIP_CANCELLED)

**Data**: 05/05/2026
**Status**: 📋 Planejado
**Tipo**: Feature / Reliability
**Tamanho**: Médio (2–3 dias)
**Prioridade**: Alta
**Depende de**: HIST-2026-037 (auto-dispatch — publica TripCreatedEvent via outbox), HIST-2026-038 (AWS SNS — NotificationServicePort real)

---

## 1. Contexto & Motivação

A infraestrutura de eventos existe e funciona parcialmente: domain events são publicados via `DomainEventPublisher`, coletados por `DomainEventsCollector`, persistidos no `outbox_event` via `JpaTripRepositoryAdapter.save()`, e processados por `OutboxPublisher` (a cada 5 s) ou `OutboxEventProcessor` (a cada 1 s). Ambos chamam `EventDispatcherPort.dispatch(eventId, eventType, payload)`.

**O problema:** `NoOpEventDispatcher` — a única implementação de `EventDispatcherPort` — ignora todos os eventos. Nenhuma notificação chega a passageiro ou motorista, independentemente do que ocorre na corrida.

Além disso, `StartTripUseCase` não publica nenhum evento ao iniciar a corrida — o evento `TripStartedEvent` não existe, tornando impossível notificar o passageiro sobre o início da viagem.

### Estado atual do loop de corrida

| Transição | Evento publicado | Handler | Notificação enviada |
|---|---|---|---|
| Passageiro cria corrida | `TripCreatedEvent` ✅ | `TripAutoDispatchHandler` (HIST-2026-037) | Motoristas via broadcast ✅ |
| Motorista aceita corrida | `TripDriverAssignedEvent` ✅ | ❌ nenhum | ❌ passageiro não é notificado |
| Motorista inicia corrida | ❌ evento não existe | ❌ | ❌ passageiro não é notificado |
| Corrida concluída | `TripCompletedEvent` ✅ | ❌ nenhum | ❌ nenhum dos dois é notificado |
| Corrida cancelada | `TripCancelledEvent` ✅ | ❌ nenhum | ❌ nenhum dos dois é notificado |

### Gaps identificados no código

**Gap 1 — `NoOpEventDispatcher` quebra o pipeline de notificações:**
```java
// infrastructure/messaging/NoOpEventDispatcher.java
@Override
public void dispatch(UUID eventId, String eventType, String payload) throws Exception {
    logger.debug("NoOpEventDispatcher: Would dispatch eventId={}, type={}", eventId, eventType);
    // No-op: does nothing in local/test environment  ← nunca foi substituído em produção
}
```

**Gap 2 — `TripStartedEvent` não existe; `StartTripUseCase` não publica evento:**
```java
// StartTripUseCase.java — ausência de publicação de evento
@Override
public Trip execute(UUID tripId) {
    Trip trip = tripRepository.findById(tripId).orElseThrow(...);
    trip.start();
    return tripRepository.save(trip);   // ← nenhum evento publicado
}
```

**Gap 3 — Eventos de corrida não carregam todos os IDs necessários para roteamento:**
- `TripDriverAssignedEvent(tripId, driverId)` — não inclui `passengerId` (necessário para notificar passageiro)
- `TripCancelledEvent(tripId, cancelledBy, reason, fee, cancelledAt)` — não inclui `passengerId` nem `driverId` (necessário para notificar ambos)
- `TripCompletedEvent(tripId, passengerId, driverId, ...)` ✅ completo

**Gap 4 — Dois schedulers concorrem pelo mesmo outbox:**
`OutboxPublisher` (a cada 5 s) e `OutboxEventProcessor` (a cada 1 s) ambos fazem `findPendingBatch` + `markSent` sem lock distribuído — um evento pode ser processado duas vezes em burst. Este gap é pré-existente e será documentado, mas a resolução definitiva (lock de linha ou `SKIP LOCKED`) é fora do escopo desta história.

## 2. Objetivo e Escopo

### 2.1 Criar `TripStartedEvent`

### 2.2 Enriquecer eventos existentes com IDs de roteamento

**`TripDriverAssignedEvent`** — adicionar `passengerId`:
```java
// ANTES
public record TripDriverAssignedEvent(String eventId, LocalDateTime occurredOn, TripId tripId, DriverId driverId)

// DEPOIS
public record TripDriverAssignedEvent(String eventId, LocalDateTime occurredOn, TripId tripId, PassengerId passengerId, DriverId driverId)
```

**`TripCancelledEvent`** — adicionar `passengerId` e `driverId` (nullable — corrida pode ser cancelada antes de ter motorista):
```java
// DEPOIS
public record TripCancelledEvent(
    String eventId, LocalDateTime occurredOn,
    TripId tripId,
    @Nullable PassengerId passengerId,
    @Nullable DriverId driverId,
    ActorType cancelledBy, CancellationReason reason, Money fee, LocalDateTime cancelledAt
)
```

**Por que enriquecer os eventos e não carregar do banco no dispatcher:**
O dispatcher processa eventos do outbox de forma assíncrona, potencialmente muito após a transação original. Carregar o trip no dispatcher introduz dependência de leitura no processamento de eventos e abre janela para race conditions (trip já pode estar em estado diferente). Eventos self-contained são mais resilientes.

### 2.3 Atualizar `StartTripUseCase` para publicar evento

```java
// application/usecases/trip/StartTripUseCase.java
public class StartTripUseCase implements StartTripInputPort {

    private final TripRepositoryPort tripRepository;
    private final DomainEventPublisher eventPublisher;

    public StartTripUseCase(TripRepositoryPort tripRepository, DomainEventPublisher eventPublisher) {
        this.tripRepository = tripRepository;
        this.eventPublisher = eventPublisher;
    }

    @Override
    public Trip execute(UUID tripId) {
        Trip trip = tripRepository.findById(tripId)
            .orElseThrow(() -> new TripNotFoundException(tripId));
        trip.start();
        Trip saved = tripRepository.save(trip);
        eventPublisher.publish(new TripStartedEvent(
            saved.getId(),
            saved.getPassengerId(),
            saved.getDriverId()   // pode ser null se não atribuído
        ));
        return saved;
    }
}
```

### 2.6 Criar `TripNotificationEventDispatcher` — substitui `NoOpEventDispatcher`

```java
// infrastructure/messaging/TripNotificationEventDispatcher.java
@Component
@Primary
@RequiredArgsConstructor
@Slf4j
public class TripNotificationEventDispatcher implements EventDispatcherPort {

    private final SendNotificationInputPort sendNotification;
    private final ObjectMapper objectMapper;

    @Override
    public void dispatch(UUID eventId, String eventType, String payload) throws Exception {
        switch (eventType) {
            case "TripDriverAssignedEvent" -> handleDriverAssigned(payload);
            case "TripStartedEvent"        -> handleTripStarted(payload);
            case "TripCompletedEvent"      -> handleTripCompleted(payload);
            case "TripCancelledEvent"      -> handleTripCancelled(payload);
            default -> log.debug("No notification handler for eventType={}", eventType);
        }
    }

    private void handleDriverAssigned(String payload) throws Exception {
        var event = objectMapper.readValue(payload, TripDriverAssignedEventPayload.class);
        if (event.passengerId() == null) return;
        sendNotification.execute(new SendNotificationCommand(
            new UserId(event.passengerId()),
            NotificationType.TRIP_ACCEPTED,
            "Motorista a caminho",
            "Seu motorista foi designado e está a caminho.",
            Map.of("tripId", event.tripId(), "driverId", event.driverId()),
            new TenantId(event.tenantId()),
            "trip-accepted-" + event.tripId()
        ));
    }

    private void handleTripStarted(String payload) throws Exception {
        var event = objectMapper.readValue(payload, TripStartedEventPayload.class);
        if (event.passengerId() == null) return;
        sendNotification.execute(new SendNotificationCommand(
            new UserId(event.passengerId()),
            NotificationType.TRIP_STARTED,
            "Corrida iniciada",
            "Sua corrida começou. Boa viagem!",
            Map.of("tripId", event.tripId()),
            new TenantId(event.tenantId()),
            "trip-started-" + event.tripId()
        ));
    }

    private void handleTripCompleted(String payload) throws Exception {
        var event = objectMapper.readValue(payload, TripCompletedEventPayload.class);
        // Notifica passageiro
        sendNotification.execute(new SendNotificationCommand(
            new UserId(event.passengerId()),
            NotificationType.TRIP_COMPLETED,
            "Corrida concluída",
            String.format("Corrida finalizada. Valor: %s", event.actualFare()),
            Map.of("tripId", event.tripId(), "fare", event.actualFare()),
            new TenantId(event.tenantId()),
            "trip-completed-passenger-" + event.tripId()
        ));
        // Notifica motorista
        sendNotification.execute(new SendNotificationCommand(
            new UserId(event.driverId()),
            NotificationType.TRIP_COMPLETED,
            "Corrida concluída",
            String.format("Corrida finalizada. Valor: %s", event.actualFare()),
            Map.of("tripId", event.tripId(), "fare", event.actualFare()),
            new TenantId(event.tenantId()),
            "trip-completed-driver-" + event.tripId()
        ));
    }

    private void handleTripCancelled(String payload) throws Exception {
        var event = objectMapper.readValue(payload, TripCancelledEventPayload.class);
        // Notifica passageiro (sempre tem)
        if (event.passengerId() != null) {
            sendNotification.execute(new SendNotificationCommand(
                new UserId(event.passengerId()),
                NotificationType.TRIP_CANCELLED,
                "Corrida cancelada",
                "Sua corrida foi cancelada.",
                Map.of("tripId", event.tripId(), "reason", event.reason()),
                new TenantId(event.tenantId()),
                "trip-cancelled-passenger-" + event.tripId()
            ));
        }
        // Notifica motorista (pode não existir se cancelado antes da atribuição)
        if (event.driverId() != null) {
            sendNotification.execute(new SendNotificationCommand(
                new UserId(event.driverId()),
                NotificationType.TRIP_CANCELLED,
                "Corrida cancelada",
                "A corrida foi cancelada pelo passageiro.",
                Map.of("tripId", event.tripId(), "reason", event.reason()),
                new TenantId(event.tenantId()),
                "trip-cancelled-driver-" + event.tripId()
            ));
        }
    }
}
```

**Observação sobre `userId` para notificações:** As notificações requerem `UserId` que é usado pelo `SnsNotificationAdapter` (HIST-2026-038) para localizar o `snsEndpointArn` via `DeviceTokenRepositoryPort`. O `UserId` neste contexto é o `keycloakId` do usuário (definido após HIST-2026-034). Os eventos de domínio contêm `passengerId` (UUID interno) — o dispatcher precisa que os payloads transportem o `keycloakId` OU que `SendNotificationUseCase` / `SnsNotificationAdapter` resolva o keycloakId a partir do `passengerId` interno.

**Solução preferida:** Os payloads dos eventos incluem `tenantId` + `passengerId` (UUID interno). O `SnsNotificationAdapter` consulta `DeviceToken` por `(tenantId, userId)` onde `userId` é o `keycloakId`. Portanto, os eventos e o dispatcher precisam usar o `keycloakId` dos usuários, não o UUID interno. Após HIST-2026-034, `Passenger.keycloakId` e `Driver.keycloakId` existem — os eventos devem transportar esses IDs ou o dispatcher deve resolvê-los via `PassengerRepositoryPort.findById()`.

**Decisão de implementação:** Incluir `keycloakId` nos payloads dos eventos quando disponível; fallback para resolução via repositório no dispatcher se `keycloakId` for nulo (caso HIST-2026-034 ainda não implementado). Isso garante que HIST-2026-039 possa ser implementado independentemente da ordem de execução de HIST-2026-034.

## 5. Fluxo Completo Após HIST-2026-039

```
Passageiro cria corrida
  → CreateTripUseCase → TripCreatedEvent → outbox
  → OutboxPublisher → TripNotificationEventDispatcher (sem handler — tipo não tratado, log debug)
  → TripAutoDispatchHandler (HIST-2026-037) → FindAvailableDrivers → NotificationServicePort.broadcast(TRIP_REQUEST)
  → SNS → FCM/APNS → motoristas próximos recebem push

Motorista aceita corrida (primeiro-a-chegar)
  → AssignDriverToTripUseCase → TripDriverAssignedEvent(tripId, passengerId, driverId) → outbox
  → OutboxPublisher → TripNotificationEventDispatcher
  → SendNotification(passageiro, TRIP_ACCEPTED) → SNS → push para passageiro ✅

Motorista inicia corrida
  → StartTripUseCase → TripStartedEvent(tripId, passengerId, driverId) → outbox
  → OutboxPublisher → TripNotificationEventDispatcher
  → SendNotification(passageiro, TRIP_STARTED) → SNS → push para passageiro ✅

Corrida concluída
  → CompleteTripUseCase → TripCompletedEvent(tripId, passengerId, driverId, fare) → outbox
  → OutboxPublisher → TripNotificationEventDispatcher
  → SendNotification(passageiro, TRIP_COMPLETED) + SendNotification(motorista, TRIP_COMPLETED) ✅

Corrida cancelada
  → CancelTripUseCase → TripCancelledEvent(tripId, passengerId, driverId, ...) → outbox
  → OutboxPublisher → TripNotificationEventDispatcher
  → SendNotification(passageiro, TRIP_CANCELLED) + SendNotification(motorista, TRIP_CANCELLED) ✅
```
