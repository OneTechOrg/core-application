# HIST-2026-022: Reativação de Motorista no Cancelamento

**Data**: 09/03/2026  
**Status**: ✅ Concluído  
**Tipo**: Correção  
**Tamanho**: Pequeno

## Problema
Ao cancelar uma corrida já atribuída, o motorista podia continuar em estado `BUSY`.

## Correção
- `CancelTripUseCase` passou a receber `DriverRepositoryPort`.
- Após cancelar a corrida, o use case reativa o motorista quando ele estiver `BUSY`.
- Adicionado `@Transactional` para manter consistência entre atualização de corrida e motorista.

## Critério de aceite
- Corrida cancelada com motorista atribuído deve deixar o motorista disponível (`ACTIVE`) quando aplicável.
