# Twilio Setup Guide for SMS OTP Onboarding

This document details the manual steps required in the Twilio Console to enable the SMS OTP Onboarding feature in production/dev environments.

---

## 1. Account Credentials

You will need your primary account credentials to authenticate the SDK:

1.  Log in to the [Twilio Console](https://www.twilio.com/console).
2.  On the Dashboard, locate the **Account SID** and **Auth Token**.
3.  Set these values in your environment:
    *   `TWILIO_ACCOUNT_SID`
    *   `TWILIO_AUTH_TOKEN`

---

## 2. Create a Verify Service

The implementation uses the **Twilio Verify API** (v2), which manages the generation, delivery, and lifecycle of OTP codes automatically.

1.  Navigate to **Verify > Services** in the Twilio Console.
2.  Click **Create Service**.
3.  **Friendly Name**: Choose a name (e.g., `RappiDrive-Auth`).
4.  Once created, copy the **Service SID** (starts with `VA...`).
5.  Set this value in your environment:
    *   `TWILIO_VERIFY_SERVICE_SID`

---

## 3. Service Configuration Recommendations

Inside your Verify Service settings, ensure the following are configured:

### A. Delivery Channels
- Enable the **SMS** channel.
- (Optional) Enable **WhatsApp** if the mobile app supports it (the backend code is currently optimized for SMS but Verify handles both).

### B. Code Length & Type
- **Code Length**: 6 digits (default and recommended).
- **Lookup**: Enabled (recommended) to validate phone numbers before sending SMS.

### C. Fraud Protection (Crucial)
To prevent "SMS Pumping" attacks and high costs:
1.  Enable **Fraud Guard** (Standard or Max).
2.  Set **Rate Limits** within the Twilio Console (separate from the backend rate limiter) to limit attempts per phone number.

---

## 4. Environment Variables Summary

For the backend to function, ensure the following variables are provided to the Spring Boot application:

| Variable | Description | Source |
| :--- | :--- | :--- |
| `TWILIO_ACCOUNT_SID` | Main Account Identifier | Dashboard |
| `TWILIO_AUTH_TOKEN` | Main Account Secret | Dashboard |
| `TWILIO_VERIFY_SERVICE_SID` | Specific Verify Service ID | Verify > Services |
| `PHONE_TOKEN_SECRET` | Secret for internal JWT (min 32 chars) | Generated by you |

---

## 5. Testing without costs

During development, you can use **Twilio Test Credentials** or defined **Mock Phone Numbers** in the Verify Service to simulate "Approved" responses without actually sending or being charged for SMS.
